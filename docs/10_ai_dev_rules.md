# KI-Entwicklungsregeln

## Workflow
1. relevante MD-Dateien lesen
2. Vorabbericht
3. nur vereinbarten Scope patchen
4. Build / Syntax / Tests
5. Abschlussbericht
6. immer direkt den nächsten Copy-Paste-Prompt liefern

## Verbote
- keine Scope-Erweiterung ohne Not
- keine stillen Refactors
- keine UI- und Pipeline-Vermischung
- keine API-Logik direkt in UI

## Pflicht
- Logging erhalten
- Crashpfade nicht entfernen
- Dokumentation bei Strukturänderungen aktualisieren
- Changelog pflegen
